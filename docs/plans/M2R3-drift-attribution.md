# M2R3 — Drift attribution: the retake gets an exit

**Trigger:** the M2 drift soak's adjudicated finding (docs/retros/M2-drift-soak.md, ruled 2026-07-20): continuity drifted delta 5.2 for 13 consecutive samples with the retake open for 21 turns — and the driver was the PLAYER (one continuous evening of inputs serializing an episodic premise). The KA following the player was correct (§0 authority ordering); the Sakkan measuring the divergence was correct; the machinery had no box for "the player did it" and no exit but retaking forever, silently.

**Ledger row.** Named failure mode: **the eternal retake** — sustained player-driven divergence has no terminal state; the engine strains against the player indefinitely and calls the doctrine working a FAIL. Pillar: §0 (player word outranks premise-truth) + §4.5 (honest measurement) + §7.1 (the Director reads whether the story still IS what it framed itself as) + §8 steering honesty ("name it — once, gently") + §4.2 (the active-layer override machinery, which already exists for exactly this shape of evolution).

**The design law (user-ratified):** the Sakkan stays blind to the DIALS — a judge who knows the target scores toward it, and the measurement's whole value is that it can't flatter. What it gains is sight of the PLAYER. Attribution is a second question, asked only when the alarm is already ringing.

## Deliverables (one commit)

1. **The attribution read.** When an axis trips the triple gate (delta ≥ 2, confidence ≥ 0.6, consecutive ≥ 2 — existing drift-band code, src/lib/sakkan/), a judgment-tier attribution probe runs ONCE per trip (not per sample): given the drifting axis, its direction, and the window's PLAYER INPUTS (with minimal narration tails for context — never the premise dials), classify the driver: `player_driven` | `narrator_driven` | `entangled`, with one evidence sentence. Cheap by construction — it fires only on gate trips, which the soak showed are rare.
2. **The routing.** `narrator_driven` (and `entangled`, conservatively) → today's behavior: the retake/amendments pressure, unchanged. `player_driven` → the retake CLOSES (the engine stops silently straining against the player) and the finding routes to the Director's next dossier as a first-class item.
3. **The exit — steering honesty + evolution.** The Director, on receiving a player-driven drift finding, may emit (via the EXISTING §4.2 arc_override path — active layer only, latest-wins, logged, canonical untouched): the axis moved to where the play actually lives, with its transition signal naming the observation in the fiction's own language. The player sees ONE quiet, dismissible notice (the assertion-notice component family): the studio noticed the story runs [X] against the premise's [Y]; it is now leaning with you — /meta to talk about it, settings/studio surfaces show the override. Silence is consent; the override is reversible; player word remains sovereign at every step.
4. **Measurement follows the effective premise** (already true — the Sakkan scores against the effective premise including overrides), so an accepted evolution RESOLVES the drift honestly instead of redefining it away.
5. **Tests:** attribution probe schema + prompt carries inputs never dials (prompt-spec pin); routing (player_driven closes the retake, narrator_driven does not); Director dossier carries the finding; the notice renders once and dismisses. Soak-harness: the verdict machinery learns the third class (`player_driven — escalated`) so the next soak cannot re-produce the M2 ambiguity.

**Out of scope:** any change to Sakkan axis scoring or its blindness; auto-evolving the CANONICAL layer (never); per-sample attribution (gate-trip only).
