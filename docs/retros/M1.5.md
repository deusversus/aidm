## What was harder than planned

- **Docker native-binary path.** First Dockerfile attempt used `COPY --from=builder` of pnpm's symlinked `@anthropic-ai/claude-agent-sdk-linux-x64-musl`. BuildKit couldn't checksum the symlink; four consecutive prod builds failed before I noticed because Railway auto-deploys on push and I stopped watching. Fix: `npm install` into a throwaway `/tmp` dir in the runner stage, then move just the one resolved package into `/app/node_modules`. Two lessons: (a) assume nothing about pnpm's layout surviving Docker COPY; (b) watch the build logs, don't assume push == deploy.
- **Railway SSE proxy buffering.** Router short-circuit turns (2 tiny events, no streaming deltas) landed only after page refresh in prod. Continue-branch turns (KA streaming) were fine because token flow kept the pipe warm. Fix: leading 2 KB-padded comment line + `Content-Encoding: identity` header. Standard SSE proxy workaround but only visible under real traffic.
- **Test migration churn.** Commit C's runner refit changed the default provider from "Google on fast" to "Anthropic everywhere." Every fast-tier consultant test that injected a `fakeGoogle` client needed flipping to `fakeAnthropic`. ~5 test files. Mechanical but real work, and the failure mode was confusing (`DATABASE_URL missing` errors because unmocked `getAnthropic()` was being called, which parsed env, which failed) before I traced the cause.
- **Voice consistency vs voice preference pushback.** Mid-session the user challenged the single-provider-lock assumption and we spent significant context in void-mode deciding between BYOK / hosted / hybrid. Landed on hosted + cost-forward + service fee, with user-selectable per-campaign provider + model within a bounded roster. The M1.5 architecture built here reflects that decision; if it had landed pre-M1.5 planning, Commit A's registry shape would have been the starting point instead of the endpoint.

## What the architecture now supports

- User flips a campaign between Opus 4.7, Sonnet 4.6, or any Opus/Sonnet snapshot in settings; next turn narrates on that model. Verified end-to-end in tests (FU-A integration test + key-animator unit tests).
- Voice consistency across a whole campaign via snapshot pinning. A writer who tunes their ear to Opus 4.5 can stay on Opus 4.5 indefinitely even as Anthropic defaults shift.
- Tier-appropriate cost discipline: fast-tier consultants automatically run on Haiku for Anthropic campaigns. Full-Opus-everywhere campaigns cost 10× a Sonnet-creative + Haiku-fast configuration.
- Silent-fallback audit trail: `resolveModelContext` logs warn when falling back to anthropic defaults (parse fail or half-migrated row), so a Sonnet-configured campaign suddenly narrating on Opus shows up in the trace.

## What M2 uses directly from M1.5

- **Provider registry + tier_models per campaign** — SZ conductor reads the campaign's modelContext via the same `resolveModelContext` pathway. Every SZ sub-agent (AnimeResearcher, ProfileResolver, ProfileGenerator, HandoffCompiler) inherits `AgentDeps.modelContext` automatically; no new wiring needed.
- **Settings UI pattern** — SZ adds a "choose provider before premise entry" step on the new-campaign flow, reusing the provider dropdown + tier-model dropdowns from `settings-ui.tsx`.
- **Per-campaign billing substrate (M2.5 dependency)** — Langfuse spans now carry `{ provider, model, tier }` metadata at every LLM call. M2.5's cost aggregation rolls up from there per-campaign, per-user. No retrofit needed.

## The one architectural decision to revisit

The provider guard in `runKeyAnimator` throws for non-anthropic providers. At M3.5 this becomes a routing decision in a new `src/lib/agents/ka/dispatch.ts` that picks `ka/anthropic.ts`, `ka/google.ts`, `ka/openai.ts`, or `ka/openrouter.ts` based on the campaign's provider. FU-D: that dispatcher refactor should be M3.5 Commit A — not a late-milestone fix. Before that dispatcher exists, "flipping `available: true`" on Google would validate config fine at settings-save but terminal-error at KA invocation. Not a regression; just the real architectural remaining-work that M3.5 needs to land first.
